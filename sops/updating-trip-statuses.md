# Updating Trip Statuses

## Purpose

To ensure trip statuses accurately reflect driver movement and load progress by cross-referencing Samsara data and updating trips in DITAT throughout the day.

## Scope

Applies to assigned personnel responsible for monitoring trips and maintaining accurate trip statuses during daily operations.

## Systems Used

- DITAT (TMS)
- Samsara

## Procedure

### Step 1: Log Into Required Systems

1.  Log into **DITAT**
2.  Log into **Samsara**
3.  Ensure both systems are open and accessible for reference

### Step 2: Review Active Trips in DITAT

1.  In DITAT, navigate to the **Trips / Dispatch / Active Trips** view
2.  Identify all trips scheduled for the current day
3.  Note current trip statuses (e.g., not started, in transit, delivered)

### Step 3: Verify Driver Movement in Samsara

1.  Switch to **Samsara**
2.  Locate the assigned driver or truck
3.  Review:
    - Current location
    - Movement status
    - Timestamp of last activity
4.  Confirm whether the driver has:
    - Started the trip
    - Arrived at pickup
    - Departed pickup
    - Arrived at delivery
    - Completed delivery

### Step 4: Update Trip Status in DITAT

Based on Samsara data:
1.  Return to **DITAT**
2.  Open the corresponding trip
3.  Update the trip status to match actual driver activity, such as:
    - Trip started
    - At pickup
    - Loaded / departed pickup
    - At delivery
    - Trip completed
4.  Save changes

### Step 5: Repeat for All Active Trips

- Continue checking each active trip
- Cross-reference Samsara movement
- Update DITAT accordingly until all trips for the day are current

### Step 6: Monitor for Exceptions

While reviewing trips, watch for:
- Drivers not moving as expected
- Delays or extended dwell times
- Status mismatches between systems
If an issue is identified:
- Notify dispatch immediately
- Document the concern if required

### Step 7: Ongoing Updates Throughout the Day

- Recheck Samsara periodically
- Update DITAT as driver statuses change
- Ensure trip statuses remain current until completion

## Error Handling

- **Samsara shows movement but DITAT is not updated**
  → Update DITAT immediately
- **DITAT shows activity but Samsara does not**
  → Verify driver assignment and escalate if needed
- **System outage or access issue**
  → Notify management and document manually if required

## Output

- All trips reflect accurate, real-time status
- Dispatch has visibility into driver progress
- Reduced risk of missed updates or miscommunication
